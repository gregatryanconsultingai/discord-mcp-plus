## Dockerfile

Multi-stage build using `node:20-alpine`.

**Build stage:**
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build
```

**Production stage:**
```dockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY --from=build /app/dist ./dist
ENV DISCORD_MCP_TRANSPORT=http
ENV DISCORD_MCP_HTTP_PORT=3000
EXPOSE 3000
CMD ["node", "dist/index.js"]
```

The container runs in HTTP mode by default ‚Äî the intended deployment pattern for Docker. `DISCORD_TOKEN` is always required at runtime and must be supplied via `-e` or a secrets manager.

## GitHub Actions ‚Äî CI

**File:** `.github/workflows/ci.yml`

Triggers on push to `master` and on pull requests targeting `master`.

Steps:
1. Checkout
2. Setup Node 20
3. `npm ci`
4. `npm test` ‚Äî all Vitest tests
5. `npm run typecheck` ‚Äî TypeScript strict check

```yaml
name: CI
on:
  push:
    branches: [master]
  pull_request:
    branches: [master]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm test
      - run: npm run typecheck
```

## GitHub Actions ‚Äî Publish

**File:** `.github/workflows/publish.yml`

Triggers on push of tags matching `v*` (e.g. `v0.8.0`).

Two parallel jobs:

### npm publish job
1. Checkout
2. Setup Node 20 with `registry-url: https://registry.npmjs.org`
3. `npm ci`
4. `npm run build`
5. `npm publish --access public`
   - Uses `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}`

### Docker publish job
1. Checkout
2. Login to Docker Hub with `docker/login-action@v3`
   - `username: ${{ secrets.DOCKER_USERNAME }}`
   - `password: ${{ secrets.DOCKER_TOKEN }}`
3. Build and push with `docker/build-push-action@v5`
   - Tags: `gregatryanconsultingai/discord-mcp-plus:latest` and `gregatryanconsultingai/discord-mcp-plus:${{ github.ref_name }}`

No new Vitest tests ‚Äî this version is pure infrastructure. The CI workflow itself is the test: it runs the existing 83-test suite on every push.

Manual smoke test before tagging `v0.8.0`:
1. `npm run build` ‚Äî verify `dist/index.js` emits correctly
2. `docker build -t discord-mcp-plus:test .` ‚Äî verify image builds
3. `docker run --rm -e DISCORD_TOKEN=fake discord-mcp-plus:test` ‚Äî verify it starts and fails gracefully on bad token (not a crash)
