# discord-mcp-plus v0.7 — HTTP Transport

**Date:** 2026-04-15
**Status:** Approved

---

## Goal

Add an HTTP transport mode so the server can be deployed as a container or remote service rather than requiring a local stdio process. `DISCORD_MCP_TRANSPORT=http` switches modes; stdio remains the default. No new tools. No new files.

---

## Architecture

| File | Change |
|------|--------|
| `src/config.ts` | Add `transport: 'stdio' \| 'http'`, `httpPort: number`, `httpToken: string \| false`. Throw at load time if `transport='http'` and no token set. |
| `src/server.ts` | Extract `createMcpServer()` helper. Branch on `config.transport` — stdio path unchanged, HTTP path creates Node HTTP server + `StreamableHTTPServerTransport`. Extract `handleHttpRequest()` for testability. |
| `src/init.ts` | Add `DISCORD_MCP_TRANSPORT`, `DISCORD_MCP_HTTP_PORT`, `DISCORD_MCP_HTTP_TOKEN` to the `.env` template. |
| `src/index.ts` | Add startup warning when HTTP mode is active. |
| `tests/config.test.ts` | 4 new tests for transport/port/token parsing. |
| `tests/server.test.ts` | New file — 3 tests for HTTP auth middleware and path routing. |

---

## New Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `DISCORD_MCP_TRANSPORT` | `stdio` | Transport mode: `stdio` or `http` |
| `DISCORD_MCP_HTTP_PORT` | `3000` | Port to listen on (HTTP mode only) |
| `DISCORD_MCP_HTTP_TOKEN` | — | Bearer token for HTTP auth. **Required** when transport is `http` — server refuses to start without it. |

## Config Changes

```typescript
// New fields in Config interface:
transport: 'stdio' | 'http'
httpPort: number          // default 3000; only meaningful when transport='http'
httpToken: string | false // false = disabled (stdio mode); required string in http mode
```

```typescript
// New parsing in loadConfig():
const transport = (env['DISCORD_MCP_TRANSPORT'] ?? 'stdio') as 'stdio' | 'http'
if (transport !== 'stdio' && transport !== 'http') {
  throw new Error("DISCORD_MCP_TRANSPORT must be 'stdio' or 'http'")
}
const httpToken = env['DISCORD_MCP_HTTP_TOKEN'] || false
if (transport === 'http' && !httpToken) {
  throw new Error('DISCORD_MCP_HTTP_TOKEN is required when DISCORD_MCP_TRANSPORT=http')
}
const httpPort = parseInt(env['DISCORD_MCP_HTTP_PORT'] ?? '3000', 10)
```

The `transport: 'stdio'` literal type in the current interface is widened to `'stdio' | 'http'`. The hardcoded `transport: 'stdio'` in `loadConfig()` is replaced with the parsed value above.

## Server Changes

### `createMcpServer(registry, client)`

Extracted from `startServer()`. Creates and configures the MCP `Server` instance — registers `ListTools` and `CallTool` handlers. Returns the configured `Server`. This makes the stdio and HTTP paths share the same server setup without duplication.

### `handleHttpRequest(req, res, config, transport)`

Standalone function handling a single HTTP request:
1. Check `Authorization: Bearer <token>` — return 401 if missing or wrong.
2. Check URL — only `/mcp` is valid, return 404 otherwise.
3. Delegate to `transport.handleRequest(req, res)`.

Extracted as a named function so it can be unit-tested without a real HTTP server.

### `startServer(registry, client, config)`

```
if config.transport === 'stdio':
  existing behavior — StdioServerTransport, server.connect(transport)

if config.transport === 'http':
  mcpServer = createMcpServer(registry, client)
  transport = new StreamableHTTPServerTransport({ sessionIdGenerator: randomUUID })
  await mcpServer.connect(transport)
  httpServer = createServer((req, res) => handleHttpRequest(req, res, config, transport))
  httpServer.listen(config.httpPort)
  log: '[discord-mcp-plus] HTTP transport listening on port <N>'
```

### MCP endpoint

`POST /mcp` and `GET /mcp` — per the MCP Streamable HTTP spec. The `StreamableHTTPServerTransport` handles session ID negotiation internally via the `Mcp-Session-Id` header.

## Testing Strategy

### `tests/config.test.ts` — 4 new tests

Append a new `describe('http transport config')` block:

1. `transport` defaults to `'stdio'` when `DISCORD_MCP_TRANSPORT` is not set
2. `DISCORD_MCP_TRANSPORT=http` parses to `'http'`
3. `DISCORD_MCP_HTTP_PORT` defaults to `3000`; custom value parses correctly
4. `loadConfig()` throws when `transport='http'` and `DISCORD_MCP_HTTP_TOKEN` is not set

### `tests/server.test.ts` — 3 new tests (new file)

Tests target `handleHttpRequest` directly — no real HTTP server, no real MCP transport.

Mock pattern: inject a fake `req` (with `headers` and `url`), fake `res` (with `writeHead` and `end` spies), and a fake `transport` (with `handleRequest` spy).

1. Missing `Authorization` header → `res.writeHead(401)` called, `transport.handleRequest` not called
2. Wrong bearer token → `res.writeHead(401)` called
3. Valid auth + `/mcp` path → `transport.handleRequest(req, res)` called

**Total new tests:** 7. Running total: 80 (73 existing + 7 new).
