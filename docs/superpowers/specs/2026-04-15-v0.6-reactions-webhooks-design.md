## Tool Specifications

### Reactions (`src/tools/reactions.ts`)

All five reaction tools share a common fetch pattern: `channel.messages.fetch(messageId)` to get the message, then operate on `message.reactions`. Emoji accepted as a plain string — Unicode directly (`👍`) or custom guild emoji as `name:id` (`party_blob:123456789`). discord.js handles both natively.

---

#### `add_reaction` (write)

Adds an emoji reaction to a message.

**Inputs:**
- `channelId` — string, required
- `messageId` — string, required
- `emoji` — string, required. Unicode emoji or `name:id` for custom emoji.

**Implementation:** `channel.messages.fetch(messageId)` → `message.react(emoji)`

**Returns:**
```typescript
{ success: true, emoji: string, messageId: string }
```

**Errors:** Throws if channel not found, message not found, or emoji is invalid.

---

#### `remove_reaction` (write)

Removes the bot's own reaction from a message.

**Inputs:**
- `channelId` — string, required
- `messageId` — string, required
- `emoji` — string, required

**Implementation:** `message.reactions.resolve(emoji)?.users.remove()`

**Returns:**
```typescript
{ success: true }
```

**Errors:** Throws if reaction not found.

---

#### `remove_user_reaction` (destructive)

Removes a specific user's reaction from a message. Requires MANAGE_MESSAGES permission.

**Inputs:**
- `channelId` — string, required
- `messageId` — string, required
- `emoji` — string, required
- `userId` — string, required

**Implementation:** `message.reactions.resolve(emoji)?.users.remove(userId)`

**Returns:**
```typescript
{ success: true }
```

**Errors:** Throws if reaction not found or bot lacks MANAGE_MESSAGES.

---

#### `remove_all_reactions` (destructive)

Removes all reactions from a message. Requires MANAGE_MESSAGES permission.

**Inputs:**
- `channelId` — string, required
- `messageId` — string, required

**Implementation:** `message.reactions.removeAll()`

**Returns:**
```typescript
{ success: true, messageId: string }
```

**Errors:** Throws if bot lacks MANAGE_MESSAGES.

---

#### `get_reactions` (read)

Lists users who reacted to a message with a specific emoji.

**Inputs:**
- `channelId` — string, required
- `messageId` — string, required
- `emoji` — string, required
- `limit` — number, optional. Default 100, max 100 (Discord API cap).

**Implementation:** `message.reactions.resolve(emoji)` → `reaction.users.fetch({ limit })` → map to user objects. Throws if `reactions.resolve(emoji)` returns null (no reactions for that emoji).

**Returns:**
```typescript
{
  emoji: string,
  count: number,
  users: Array<{ id: string, username: string }>
}
```

**Errors:** Throws if no reactions exist for the specified emoji on that message.

### Webhooks (`src/tools/webhooks.ts`)

---

#### `create_webhook` (write)

Creates a webhook in a channel.

**Inputs:**
- `channelId` — string, required
- `name` — string, required. Display name of the webhook.
- `avatarUrl` — string, optional. URL of avatar image.

**Implementation:** `channel.createWebhook({ name, avatar: avatarUrl })`

**Returns:**
```typescript
{ id: string, name: string, url: string, channelId: string }
```

**Errors:** Throws if bot lacks MANAGE_WEBHOOKS, channel not found, or name is invalid.

---

#### `list_webhooks` (read)

Lists webhooks for a channel or guild. If `channelId` is provided, returns webhooks for that channel. Otherwise returns all webhooks for the guild (requires `guildId` from arg or config).

**Inputs:**
- `channelId` — string, optional
- `guildId` — string, optional. Falls back to `config.guildId`.

**Implementation:**
- With `channelId`: `channel.fetchWebhooks()` via `client.channels.fetch(channelId)`
- Without `channelId`: `guild.fetchWebhooks()` via `client.guilds.fetch(guildId)`
- Throws if neither `channelId` nor `guildId` (arg or config) is available.

**Returns:**
```typescript
{
  webhooks: Array<{
    id: string,
    name: string,
    url: string,         // full webhook URL including token
    channelId: string,
    type: string         // 'incoming' | 'channel_follower' | 'application'
  }>
}
```

---

#### `delete_webhook` (destructive)

Deletes a webhook by ID.

**Inputs:**
- `webhookId` — string, required

**Implementation:** `client.fetchWebhook(webhookId)` → `webhook.delete()`

**Returns:**
```typescript
{ success: true, deletedId: string }
```

**Errors:** Throws if webhook not found or bot lacks MANAGE_WEBHOOKS.

---

#### `send_webhook_message` (write)

Sends a message via a webhook URL. Does not require the bot token — uses the webhook URL directly. Useful for CI/CD-style notifications or sending as a custom username/avatar.

**Inputs:**
- `webhookUrl` — string, required. Full webhook URL including token.
- `content` — string, required. Message text.
- `username` — string, optional. Override display name for this message.
- `avatarUrl` — string, optional. Override avatar URL for this message.

**Implementation:** `new WebhookClient({ url: webhookUrl }).send({ content, username, avatarUrl })`

**Returns:**
```typescript
{ messageId: string, channelId: string }
```

**Errors:** Throws if URL is invalid or webhook has been deleted.

## Testing Strategy

### `tests/reactions.test.ts` (6 tests)

Mock pattern: `channel.messages.fetch(messageId)` returns a fake message object with a `reactions` mock (resolve, removeAll). `reactions.resolve(emoji)` returns a fake reaction with a `users` mock (remove, fetch).

1. `add_reaction` — calls `message.react(emoji)`, returns `{ success: true, emoji, messageId }`
2. `remove_reaction` — calls `reactions.resolve(emoji).users.remove()` with no userId
3. `remove_user_reaction` — calls `reactions.resolve(emoji).users.remove(userId)` with the specified userId
4. `remove_all_reactions` — calls `message.reactions.removeAll()`, returns `{ success: true, messageId }`
5. `get_reactions` — calls `reactions.resolve(emoji).users.fetch({ limit })`, returns `{ emoji, count, users: [{id, username}] }`
6. `get_reactions` with unknown emoji — `resolve` returns null, throws descriptive error

### `tests/webhooks.test.ts` (6 tests)

Mock pattern: inject a mock `client` with `channels.fetch`, `guilds.fetch`, `fetchWebhook` spies. Mock `WebhookClient` constructor via vi.mock or inline mock object.

1. `create_webhook` — calls `channel.createWebhook({ name, avatar })`, returns `{ id, name, url, channelId }`
2. `list_webhooks` with `channelId` — uses `channel.fetchWebhooks()`, returns mapped webhook array
3. `list_webhooks` without `channelId` — uses `guild.fetchWebhooks()` with config guildId
4. `list_webhooks` without `channelId` or config guildId — throws
5. `delete_webhook` — calls `client.fetchWebhook(webhookId)` then `webhook.delete()`, returns `{ success: true, deletedId }`
6. `send_webhook_message` — constructs `WebhookClient({ url })` and calls `.send({ content, username, avatarUrl })`, returns `{ messageId, channelId }`

**Total new tests:** 12. Running total: 72 (60 existing + 12 new).
